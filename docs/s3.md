# S3 TUI Usage

A dedicated S3 browser with bucket listing, object navigation, metadata viewing, and optional delete support.

```bash
./bin/aws_explorer s3 [flags]
```

### S3 Flags

The global `--profile`, `--auth-method`, `--role-arn` and `--region` flags
apply.

| Flag | Default | Description |
|------|---------|-------------|
| `--bucket` | — | Bucket name to open directly |
| `--prefix` | — | Key prefix to start browsing from |
| `--theme` | `spotted-pardalote` | UI theme name |
| `--allow-delete` | `false` | Enable object deletion |
| `--endpoint-url` | — | Custom endpoint (LocalStack, MinIO, etc.) |

Press `o` anywhere in the browser to open the selection in the AWS console —
the current bucket on the bucket list, the bucket + prefix or the selected
object in the object list. The URL is copied to the clipboard, and a browser
opens when the session is local.

### Finding a row quickly (`:` jump)

In both the bucket list and the object list, press `:` to open a **jump**
prompt, then start typing. The highlight moves to the first row whose name
**contains** what you've typed (case-insensitive), updating on every keystroke —
like a type-ahead in a dropdown. Nothing is filtered out: the full listing stays
put and only the cursor moves.

| Key | Action |
|-----|--------|
| `:` | Open the jump prompt |
| (type) | Move the highlight to the first matching row |
| `↑` / `↓` | Step to the previous / next match |
| `Enter` | Keep the current position and close the prompt |
| `Esc` | Close the prompt |

This is separate from `/`, which keeps its existing meaning — filter on the
bucket list, edit the path/prefix on the object list.

### Previewing objects

Press `p` on an object to preview it. A CSV/TSV (or any delimited text, via `t`)
opens a full-screen **table** view; a `.parquet`/`.pq` file is decoded and its
rows shown in the same table; a `.gz` is decompressed and shown by its inner
type; a `.tar`/`.tar.gz` opens a member browser.

The table view zebra-stripes alternate rows for readability and supports:

| Key | Action |
|-----|--------|
| `↑/↓` `PgUp/PgDn` `g/G` | Scroll rows |
| `Enter` | Show the selected row vertically as `Col : value` pairs |
| `←/→` | Scroll columns (the first column stays pinned) |
| `s` / `S` | Cycle / type the delimiter (CSV only) |
| `h` | Set the header row (CSV only) |
| `n` | Set how many rows to read (Parquet only) |
| `w` | Cycle the first-N/last-N row window |
| `y` | **Copy the visible table to the clipboard as a Markdown table** (`\|col1\|col2\|`) — pipes/newlines in cells are escaped |
| `t` | Toggle to the raw-text preview (CSV only) |
| `Esc` | Close |

#### Parquet

Parquet files carry their own schema, so the table uses the column names from
the file — there is no delimiter or header-row to set. The first **1000** rows
are read by default; press `n` to read a different number. Only the file footer
and the row groups holding the requested rows are fetched, so a large object is
previewed without downloading it whole.

Copy-as-markdown copies exactly the rows currently shown (so the `w` window is
honoured), making it easy to paste a slice of a CSV into a PR or doc.

### S3 Examples

```bash
# Browse all buckets
./bin/aws_explorer s3

# Open a specific bucket
./bin/aws_explorer s3 --bucket my-bucket --region us-east-1

# Browse with a prefix filter
./bin/aws_explorer s3 --bucket my-bucket --prefix logs/2024/

# Use a different theme
./bin/aws_explorer s3 --theme oriole

# Enable deletion (use with caution)
./bin/aws_explorer s3 --bucket my-bucket --allow-delete

# Point to a local MinIO or LocalStack instance
./bin/aws_explorer s3 --endpoint-url http://localhost:9000
```
